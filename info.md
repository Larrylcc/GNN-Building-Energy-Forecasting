# 方案说明文档要求

方案说明文档需要以Markdown格式编写，文件命名为方案名称.md，存放在该方案文件夹中。

在每一个方案说明文档中，需要包含以下内容：
1. 方案的核心思想和方法。
2. 方案的实现步骤和流程。
    需要包括：源数据的预处理和数据选择方法，如：对缺失值的处理方法，对异常值的处理方法，数据归一化的具体处理方法（如有）；模型的选择和训练方法，如：选择的模型类型，模型对数据的处理过程， 模型的超参数设置，模型的训练过程和迭代次数等；模型的评估方法，如：使用的评估指标，评估数据集的划分方法等。
3. 方案的优缺点分析。
4. 方案的适用场景和限制条件。
5. 方案的实验结果和性能评估。
